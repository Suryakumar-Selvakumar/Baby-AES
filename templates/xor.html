Bitwise XOR Operations

A bitwise XOR is a binary operation that takes two bit patterns of equal length and performs the logical exclusive OR operation on each pair of corresponding bits. The result in each position is 1 if only one of the bits is 1, but will be 0 if both are 0 or both are 1. In this we perform the comparison of two bits, being 1 if the two bits are different, and 0 if they are the same.

For example:-

\[0\;1\;0\;1\quad(decimal\;5)\]

\[\quad XOR\quad0\;0\;1\;1\quad(decimal\;3)\]

\[=\quad0\;1\;1\;0\quad(decimal\;6)\]

The bitwise XOR may be used to invert selected bits in a register (also called toggle or flip). Any bit may be toggled by XORing it with 1. For example, given the bit pattern 0010 (decimal 2) the second and fourth bits may be toggled by a bitwise XOR with a bit pattern containing 1 in the second and fourth positions.

For example:-

\[0\;0\;1\;0\quad(decimal\;2)\]

\[\quad XOR\quad1\;0\;1\;0\quad(decimal\;10)\]

\[=\quad1\;0\;0\;0\quad(decimal\;8)\]

Mathematical equivalent and Truth Table

This equation defines the bitwise exclusive OR (XOR) operation between two binary numbers, \(x\) and \(y\). Assuming \(x\geq y\) for the non-negative integers, the bitwise operations can be written as follows:-

\[ x \text{ XOR } y = \sum_{n=0}^{\left\lfloor \log_2(x) \right\rfloor} 2^n \left( \left( \left( \left\lfloor \frac{x}{2^n} \right\rfloor \text{ mod } 2 \right) + \left( \left\lfloor \frac{y}{2^n} \right\rfloor \text{ mod } 2 \right) \right) \text{ mod } 2 \right) \]

\[= \sum_{n=0}^{\left\lfloor \log_2(x) \right\rfloor} 2^n \left( \left( \left\lfloor \frac{x}{2^n} \right\rfloor + \left\lfloor \frac{y}{2^n} \right\rfloor \right) \text{ mod } 2 \right) \]

In essence, this equation is a verbose and mathematical way to describe the process of taking two binary numbers, comparing each bit, performing XOR on each bit, and then combining these results to form a new binary number that represents the XOR of the original two numbers.
